# Demo Style Guide

House rules that shape how Claude Notebook writes study notes. Point the plugin's **style-guide** setting at this file and every generated note will follow these conventions.

- Always show the formula **before** plugging in numbers.
- Define every symbol on first use (e.g. "$\sigma$ is the standard deviation").
- Prefer a worked example with real numbers over abstract prose.
- Flag exam traps with a `> [!warning]` callout.
- Keep each note tight — enough to be useful, short enough to skim.
- End with a `## Key Takeaways` checklist and a `## Links` footer.
